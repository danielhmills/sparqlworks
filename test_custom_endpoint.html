Test Custom Endpoint Handling

This test demonstrates how custom SPARQL endpoints are handled and how describe bases are derived from them.

Scenario Description Inputs Actual Endpoint Expected Endpoint Actual Describe Base Expected Describe Base Status

Key Behaviors:

  • Priority: Input field > localStorage > application origin > hardcoded fallback
  • Describe Base: Always derived from the endpoint's origin (protocol + hostname + port)
  • Path Handling: Endpoint paths are preserved, but describe base uses only the origin
  • Dynamic Updates: When endpoint changes, describe base is automatically updated
  • Persistence: Custom endpoints and describe bases are saved to localStorage
